# Architecture

## High-Level Design

Monolithic full-stack application — React SPA served by a Hono API server. Single Docker image runs both the web server and (optionally) a Graphile Worker process for background jobs.

```
┌─────────────────┐     ┌──────────────────┐     ┌──────────────┐
│  React SPA      │────▶│  Hono API Server │────▶│  PostgreSQL  │
│  (Vite build)   │     │  (Node.js 20)    │     │  (Neon)      │
└─────────────────┘     └──────┬───────────┘     └──────────────┘
                               │
                        ┌──────▼───────────┐
                        │  Graphile Worker  │
                        │  (background jobs)│
                        └──────┬───────────┘
                               │
                        ┌──────▼───────────┐
                        │  OpenAI / Claude  │
                        │  (LLM grading)   │
                        └──────────────────┘
```

## Frontend Architecture

**Stack**: Vite 7 + React 19 + TanStack Router (file-based) + TanStack Query + Tailwind CSS 4 + Headless UI

### Routing
- File-based routes in `src/client/routes/`
- Route tree auto-generated by `@tanstack/router-plugin/vite`
- Role-based guards via `beforeLoad` in route definitions
- Layout routes: `__root.tsx` (Sidebar), `student.tsx`, `staff.tsx`

### State Management
- **Auth**: React Context (`AuthContext.tsx`) — stores Supabase user + DB user with role
- **Server state**: TanStack Query (5-min stale time, no refetch on window focus)
- **Local state**: Custom hooks per feature (e.g., `useAssignmentData`, `useStaffCourse`)

### API Client
- `apiClient()` in `src/client/lib/api.ts` — auto-injects Bearer token
- Dual auth: checks localStorage for custom JWT first, falls back to Supabase session
- 401 retry: attempts token refresh via `/api/auth/refresh` before failing
- Resource-specific API objects: `coursesApi`, `assignmentsApi`, `submissionsApi`, etc.

### UI Libraries
- **Headless UI** for accessible modals, menus, transitions
- **Heroicons** for icons
- **Recharts** for analytics charts
- **React Flow** (`@xyflow/react`) for UML diagram editor
- **react-markdown** for rendering markdown content

## Backend Architecture

**Stack**: Hono + Node.js + Drizzle ORM + Graphile Worker

### Request Pipeline
```
Request → Security Headers → CORS → Body Limit → Rate Limiter → Auth Middleware → RLS Middleware → Route Handler
```

### Middleware
1. **`secureHeaders`** — CSP, HSTS, X-Frame-Options (stricter in production)
2. **CORS** — restricted to `VITE_APP_URL`
3. **`bodyLimit`** — 1MB global, 10MB for CSV upload
4. **Rate limiter** — 1000 req/15min per IP
5. **`authMiddleware`** — validates JWT (custom or Supabase), attaches `user` to context
6. **`rlsMiddleware`** — sets PostgreSQL session variables for row-level security

### Auth Flow
- Dual JWT support: custom password-based JWT (via `jose`) + Supabase JWT
- JWT contains `sub` (user ID), `email`, `role`
- Refresh token rotation stored in `refreshTokens` table
- Role determination: email pattern matching for NTU domains + DB lookup

### Route Organization
11 route modules registered on the Hono app:
- `auth`, `courses`, `assignments`, `submissions`, `questions`, `tags`
- `auto-grade`, `notifications`, `settings`, `users`, `admin`

### Background Jobs (Graphile Worker)
- **`auto-grade-written`** — LLM-based essay grading with structured output
- **`auto-grade-uml`** — Vision API for UML diagram assessment
- Concurrency: 1 (sequential processing)
- Retry with exponential backoff for transient LLM errors (429, 5xx)

## Database Schema (13 tables)

```
users ─┬─ enrollments ─── courses
       │                    │
       │              assignments ─── assignmentQuestions ─── questions ─── rubrics
       │                    │
       └── submissions ─── answers ─── marks
                              │
                        aiGradingJobs

systemSettings    staffNotifications    aiUsageStats
passwordResetTokens    refreshTokens
```

### Key Design Decisions
- **JSONB** for question content, rubrics, tab switch logs — flexible schema per question type
- **Course-scoped roles** via `enrollments.role` (lecturer, ta, lab_exec, student) separate from global `users.role`
- **Submission lifecycle**: `draft → submitted → grading → graded` (also `late`)
- **AI job tracking**: per-answer granularity with batch grouping, token/cost accounting
- **Daily aggregated AI usage stats** by provider + model

## LLM Integration

- **Provider factory** (`src/server/lib/ai.ts`) — returns OpenAI or Anthropic model via Vercel AI SDK
- **Configurable at runtime** — provider/model stored in `systemSettings` table, changeable by admin
- **Settings cache** — 1-minute TTL to avoid DB reads per request
- **Structured output** — Zod schemas for parsing LLM grading responses
- **Retry logic** — exponential backoff for transient errors only (429, 5xx, network)
- **Cost tracking** — per-job token counts + cost stored in `aiGradingJobs`, aggregated daily

## Security Measures

- Security headers (CSP, HSTS, X-Frame-Options, nosniff)
- CORS restricted to frontend origin
- Rate limiting (1000 req/15min)
- Body size limits (1MB JSON, 10MB CSV)
- Non-root Docker user
- Environment validation at startup (fail-fast)
- JWT refresh token rotation (single-use tokens)
